# claude-mem 用途说明

## GitHub 地址

- [https://github.com/thedotmack/claude-mem/tree/main](https://github.com/thedotmack/claude-mem/tree/main)

## 项目定位

`claude-mem` 是一个面向 Claude Code 的持久化记忆插件。  
它的核心用途是：**把 AI 编程会话中的关键上下文长期保存，并在后续会话中按需注入**，减少重复说明与上下文丢失。

## 主要解决的问题

- 会话断开、重启后，AI 无法自然延续之前的工作上下文
- 历史问题与修复过程难以检索，容易重复排查
- 长期项目中的知识（决策、Bug 修复、实现细节）沉淀不足
- 直接注入全部历史信息成本高，Token 消耗大

## 核心用途（你可以把它当作什么）

- 一个“AI 编程记忆层”：跨会话保留工作记忆
- 一个“项目经验库”：把历史观察、总结、决策结构化存储
- 一个“可检索时间线”：支持按主题/类型/时间回溯开发过程
- 一个“上下文压缩器”：先索引后展开，降低上下文注入成本

## 关键能力概览

- 自动捕获会话中的工具调用与关键信息
- 使用 AI 进行语义压缩与摘要
- 通过搜索工具分层检索（`search` → `timeline` → `get_observations`）
- 在新会话中自动提供相关历史上下文
- 本地 Web Viewer（默认 `http://localhost:37777`）用于查看与调试记忆数据
- 支持对敏感内容进行排除控制（隐私保护）

## 典型使用场景

- 持续迭代同一个代码库时，减少“重新解释背景”的成本
- 多天/多周修复复杂问题时，快速回看曾经尝试与结论
- 团队协作中沉淀可追溯的 AI 辅助开发知识
- 需要在 Token 成本可控前提下，引入长期项目上下文

## 一句话总结

`claude-mem` 的用途是：**让 AI 编程助手具备“跨会话记忆”和“可检索项目历史”，从而提升连续开发效率与上下文一致性。**
